  Section #1: Result Retrieval

Once submitted, a temporary page will be displayed on the web page, showing the JOB_ID and indicating that the query is being processed.


  Primerize is running...


Your JOB_ID is: 0123456789abcdef





Your query is being processed. Usually, the calculation is finished within 30 seconds. Depending on the input sequence length and complexity, the run may take longer.

You can close the browser and retrieve the result later using the above unique JOB_ID. The cached result expires after 9 months.

Please this link:
http://primerize.stanford.edu/result/?job_id=0123456789abcdef


When the calculation is completed, the result page will automatically be shown. The run results are retrievable through the   Home  page with the unique 16-digit hexadecimal JOB_ID.


  Result retrieval

Previous result is retrievable by its unique JOB_ID before expiration (9 months).

For long inputs, we encourage the user to   Download  the source code and run locally, which enables more options. If the server encounters an internal error while processing the run, the following page will be shown.


  Primerize has difficulty in your query...


Primerize encountered an internal error while processing your query. Sorry for the inconvenience.

We will investigate and fix the problem.

For further information, please feel free to us to track down the problem.

You might encounter a FAILURE case.

This happens when Primerize was not able to find a solution using the given input sequence in combination with specified advanced options. The following page will be shown:


  FAILURE: No solution found (Primerize run finished without errors).

  • Please examine the advanced options. Possible solutions might be restricted by stringent options combination, especially by minimum Tm and # number of primers. Try again with relaxed the advanced options.
  • Certain input sequence, e.g. polyA or large repeats, might be intrinsically difficult for PCR assembly design.
  • For further information, please feel free to us to track down the problem.
